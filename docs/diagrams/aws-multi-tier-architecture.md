# Архітектурний Контекст (Textual Architecture Context)

Цей документ містить текстовий опис референсної архітектури (`aws-multi-tier-architecture.drawio`), який оптимізовано для використання як **системний промпт (System Prompt Context)** для LLM-аудиторів. Він допомагає моделям зрозуміти топологію мережі та взаємозв'язки ресурсів при аналізі CSPM-алертів.

---

## 🏗️ Опис Архітектури

**Назва:** AWS Multi-Tier Microservices Data Platform
**Провайдер:** AWS
**Парадигма:** 3-Tier Architecture (Public, Private, Isolated)

### 1. Периметр та Доставка Контенту (Edge & CDN)
- **External Users** підключаються виключно через протокол `HTTPS`.
- Точкою входу є **Amazon CloudFront** (CDN для кешування статики та маршрутизації запитів).
- На CloudFront накладено **AWS WAF** (Web Application Firewall) для захисту від веб-експлойтів (OWASP Top 10, DDoS).

### 2. Мережевий рівень (Networking & Subnets)
Інфраструктура розгорнута в єдиному VPC з поділом на 3 типи підмереж (Multi-AZ):

#### 2.1. Public Subnet (Публічна підмережа)
- **Internet Gateway (IGW)** забезпечує зв'язок зі світом.
- **NAT Gateway** використовується для безпечного доступу приватних ресурсів до інтернету (тільки вихідний трафік).
- **Application Load Balancer (ALB)** приймає трафік від CloudFront і рівномірно розподіляє його між мікросервісами. ALB є єдиним ресурсом в інфраструктурі (окрім CloudFront/API Gateway), який має доступ ззовні.

#### 2.2. Private Subnet (Приватна підмережа - Compute Tier)
- Тут розгорнутий кластер **Amazon EKS** (Kubernetes), який виконує бізнес-логіку мікросервісів.
- Вузли (EC2-інстанси) EKS використовують **Amazon EBS** для персистентного зберігання (persistent node storage).
- **Важливо:** Ресурси в цій мережі не мають публічних IP-адрес і не приймають трафік безпосередньо з інтернету. Весь вхідний трафік йде ТІЛЬКИ через ALB.

#### 2.3. Isolated Subnet (Ізольована підмережа - Data Tier)
- **Абсолютно ізольована** мережа. Не має маршрутів навіть до NAT Gateway (немає вихідного інтернет-доступу).
- Зберігає найважливіші бази даних: **Amazon RDS** (Multi-AZ Relational Database).
- Доступ до RDS дозволений виключно з Private Subnet (від воркерів EKS).

### 3. Асинхронний обмін та Стримінг подій (Async Messaging & Real-Time Data Pipeline)
Мікросервіси в EKS генерують та споживають події через керовані сервіси:
- **Amazon SQS** для виконання фонових задач (background tasks).
- **Amazon SNS** для розсилки повідомлень (event notifications).
- **Kinesis Data Streams** приймає потік сирих даних в реальному часі (event ingestion).
- **AWS Lambda** використовується для трансформації потоку даних на льоту (stream transform) перед тим як зберегти їх.

### 4. Зберігання та Аналітика (Storage & Analytics)
- Згенеровані та трансформовані дані зберігаються в **Amazon S3** (Data Lake). Відкритий публічний доступ до бакетів ЗАБОРОНЕНИЙ, за виключенням бакетів для CloudFront.
- **AWS Glue** слугує як Data Catalog (каталогізація схем даних S3).
- **Amazon Athena** підключається до S3/Glue для виконання Ad-hoc SQL запитів аналітиками.
- **Amazon OpenSearch** використовується для індексування логів і подій та виведення їх на Real-time дашборди.
