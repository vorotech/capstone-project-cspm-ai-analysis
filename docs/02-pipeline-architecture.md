# Архітектура Пайплайну

Цей документ описує етапи роботи інструментарію пайплайну. Це **набір автоматизованих інструментів для поетапного запуску**.

Зазвичай процес передбачає одноразове створення інфраструктури, разовий збір результатів сканерів і **багаторазовий запуск етапу LLM-аналізу** для акумулювання статистично значущих даних.

## Етапи роботи

1. **Підготовка (Git Checkout)**: Клонування репозиторію `https://github.com/epam/ecc-aws-rulepack` та виділення необхідних конфігурацій (`red` — з вразливостями, `green` — безпечних).
2. **Розгортання інфраструктури (Terraform Apply)**: Локальний запуск вибраних тестових конфігурацій для створення AWS ресурсів.
3. **Збір сирих даних (CSPM Аналіз)**:
   - Запуск **Prowler** та збереження результатів (JSON).
   - Збір результатів з **AWS Security Hub** (JSON).
4. **Підготовка контексту та Анонімізація (Data Sanitization)**: Формування «квантів високої щільності» для LLM. На цьому кроці діє механізм захисту конфіденційності — всі 12-значні AWS Account ID замінюються на маску `<REDACTED>`. Дані, що перетинають межу довіри (Trust Boundary), очищені від ідентифікаторів інфраструктури.
5. **LLM Аналіз**: Обробка анонімізованих результатів через різні LLM (за допомогою OpenRouter API). 
   - *Примітка*: Цей етап запускається багаторазово (вручну або циклічно) з різними моделями, щоб сформувати великий масив історичних даних.
   - Результати накопичуються у файлах `metrics_history.csv` та `findings_history.csv`.
6. **Аналіз результатів**: Порівняння рішень моделей, підрахунок метрик консистентності та відсотка зміни рішень (Adjustment Rate) за допомогою Jupyter Notebook.
7. **Презентація**: Візуалізація результатів у React-застосунку на основі згенерованих даних.
8. **Очищення (Terraform Destroy)**: Видалення створених AWS ресурсів для зачистки середовища та уникнення зайвих витрат (виконується в кінці циклу тестування).
